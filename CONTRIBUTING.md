# Contributing

Thank you for your interest in this repository. This is a **personal
curriculum vitae** maintained by Khaalis Wooden, so contribution works a
little differently than a typical open-source software project.

## What This Repository Is

This repo hosts a living, version-controlled professional CV, split across:

- **[README.md](./README.md)** — Overview and navigation
- **[CURRENT_FOCUS.md](./CURRENT_FOCUS.md)** — Active roles and projects
- **[VISION_FORWARD.md](./VISION_FORWARD.md)** — Strategic objectives
- **[PAST_PERFORMANCE.md](./PAST_PERFORMANCE.md)** — Career history and results

Because the content is biographical, only the repository owner can add or
change factual career information. However, feedback and corrections are
genuinely welcome.

## How You Can Help

### Reporting an issue

Open a GitHub issue if you spot any of the following:

- **Typos, grammar, or formatting** — broken Markdown, dead links, rendering
  glitches.
- **Broken navigation** — internal links between the CV documents that no
  longer resolve.
- **Accessibility** — content that is hard to read with assistive technology.

Please include the file name and a short description. A suggested fix is
appreciated but not required.

### Suggesting a change

For anything beyond a trivial typo, open an issue first to discuss the
change before submitting a pull request. Factual updates to employment,
education, or credentials will only be accepted from the owner.

## Pull Request Guidelines

If you submit a PR for a formatting or link fix:

1. Keep the change focused and small — one fix per PR.
2. Do **not** alter biographical facts, metrics, or claims.
3. Preserve the existing tone, heading structure, and emoji conventions.
4. Confirm all internal links still resolve after your change.
5. Write a clear PR description explaining what and why.

## Style Conventions

- Documents use standard GitHub-Flavored Markdown.
- Section headers use emoji prefixes (e.g., `## 🎯 Professional Summary`) —
  match the surrounding style when editing.
- Horizontal rules (`---`) separate major sections.
- Dates are written out (e.g., "May 1, 2026").
